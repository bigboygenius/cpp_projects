# Часть 1.

Класс `StackAllocator<typename T, size_t N>`, который бы позволял создавать стандартные контейнеры на стеке, без единого обращения к динамической памяти. Для этого написа класс `StackStorage<size_t N>`, в котором будет храниться большой массив на стеке. Объект класса `StackAllocator` легковесный, легко копируется и присваивается. Объект `StackStorage`, напротив, не поддерживает копирование.

Класс `StackAllocator` удовлетворяет требованиям к аллокатору, описанным на странице <https://en.cppreference.com/w/cpp/named_req/Allocator>. Он STL-совместимый, то есть позволяет использование в качестве аллокатора для 
стандартных контейнеров. В частности, определены:

- Конструктор копирования, деструктор, оператор присваивания;
- Методы `allocate`, `deallocate`;
- Внутренний тип `value_type`;

Во время исполнения может существовать несколько `StackStorage` с одним и тем же `N`. Разумеется, аллокаторы, построенные на разных `StackStorage`, должны считаться неравными.

**StackAllocator заботится о выравнивании объектов.** В частности, нельзя класть переменные типа `int` по адресам, не кратным 4; переменные типа `double` — по адресам, не кратным 8, и т.д.

Использование любого контейнера со `StackAllocator` вместо `std::allocator` приводит к тому, что обращений к динамической памяти в программе не происходит вообще.

# Часть 2.

Класс `List` — двусвязный список с **правильным использованием аллокатора**. Правильное использование аллокатора означает, что лист удовлетворяет требованиям <https://en.cppreference.com/w/cpp/named_req/AllocatorAwareContainer>. Должно быть два шаблонных параметра: `T` — тип элементов в листе, `Allocator` — тип используемого аллокатора (по умолчанию — `std::allocator<T>`).

Поддержаны:

- Конструкторы: без параметров; от одного числа; от числа и `const T&`; от одного аллокатора; от числа и аллокатора; от числа, `const T&` и аллокатора. Если создается пустой лист, то не должно быть никаких выделений динамической памяти, независимо от того, на каком аллокаторе построен этот лист.
- Метод `get_allocator()`, возвращающий объект аллокатора, используемый в листе на данный момент;
- Конструктор копирования, деструктор, копирующий оператор присваивания;
- Метод `size()`, работающий за `O(1)`, и `empty()`.
- Методы `push_back`, `push_front`, `pop_back`, `pop_front`;
- Двунаправленные итераторы, удовлетворяющие требованиям <https://en.cppreference.com/w/cpp/named_req/BidirectionalIterator>. Также поддержите константные и `reverse`-итераторы;
- Методы `begin`, `end`, `cbegin`, `cend`, `rbegin`, `rend`, `crbegin`, `crend`;
- Методы `insert(iterator, const T&)`, а также `erase(iterator)` — для удаления и добавления одиночных элементов в список.

**Если аллокатор является структурой без каких-либо полей, то такой аллокатор не должен увеличивать `sizeof` объекта листа.** То есть тривиальный аллокатор как поле листа должен занимать 0 байт.

Все методы листа строго безопасны относительно исключений. Это означает, что при выбросе исключения из любого метода класса (кроме деструктора) во время произвольной операции `X` над листом лист возвращается в состояние, которое было до начала выполнения `X`, не допустив `UB` и утечек памяти, и пробрасывает исключение наверх в вызвавшую функцию. Можно считать, что конструкторы и операторы присваивания у аллокаторов исключений никогда не кидают (это является частью требований к `Allocator`).
